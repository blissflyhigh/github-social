# README Templates and Badge Placement Patterns

Templates for badge placement and README structure that maximize engagement and clarity.

## Badge Placement Patterns

### Pattern 1: Compact Header (Recommended)

All badges on a single line or two, immediately after the H1:

```markdown
# Project Name

[![License](badge)](link) [![Version](badge)](link) [![Build](badge)](link) [![Downloads](badge)](link)

Project description starts here...
```

**Best for**: Projects with 4-8 badges, clean appearance

### Pattern 2: Categorized Rows

Badges grouped by category on separate lines:

```markdown
# Project Name

[![Build](badge)](link) [![Coverage](badge)](link) [![Quality](badge)](link)

[![Version](badge)](link) [![License](badge)](link) [![Downloads](badge)](link)

[![Stars](badge)](link) [![PRs Welcome](badge)](link) [![Discord](badge)](link)

Project description starts here...
```

**Best for**: Projects with 9+ badges, organized appearance

### Pattern 3: Centered Badges

Badges centered using HTML (more visual impact):

```markdown
# Project Name

<p align="center">
  <a href="link"><img src="badge" alt="License"></a>
  <a href="link"><img src="badge" alt="Version"></a>
  <a href="link"><img src="badge" alt="Build"></a>
</p>

Project description starts here...
```

**Best for**: Projects wanting maximum visual impact

### Pattern 4: Badge + Tagline Combo

Badges followed by a styled tagline:

```markdown
# Project Name

[![License](badge)](link) [![Version](badge)](link) [![Build](badge)](link)

> A powerful tool that does amazing things for developers

Project description starts here...
```

**Best for**: Projects with strong value proposition

## Badge Section Detection

When updating existing READMEs, detect badge sections using these patterns:

### Detection Rules

1. **After H1**: Lines starting with `[![` immediately after `# ` heading
2. **Image links**: Pattern `[![text](image-url)](link-url)`
3. **Badge URLs**: Contains `shields.io`, `badge`, `img.shields.io`
4. **Consecutive lines**: Multiple badge lines without blank lines between

### Regex Patterns

```regex
# Detect badge line
^\[!\[.*?\]\(https?://[^\)]*(?:shields\.io|badge)[^\)]*\)\]\([^\)]+\)

# Detect badge section start (after H1)
^#\s+.+\n\n(\[!\[.+$\n?)+

# Detect shields.io URL
https?://img\.shields\.io/[^\s\)]+
```

### Preservation Rules

When updating badges:
1. Identify managed badges (generated by this skill)
2. Identify custom badges (user-added, not matching patterns)
3. Replace managed badges with new set
4. Preserve custom badges at end of badge section
5. Maintain badge order: status → version → metrics → custom

## Badge Section Markers (Optional)

For easier management, use HTML comments to mark badge sections:

```markdown
# Project Name

<!-- badges-start -->
[![License](badge)](link) [![Version](badge)](link)
<!-- badges-end -->

<!-- custom-badges-start -->
[![Custom](badge)](link)
<!-- custom-badges-end -->
```

## Responsive Image Sizing

For infographics that display well on all devices:

```markdown
<!-- Full width on desktop, contained on mobile -->
<p align="center">
  <img src=".github/readme-infographic.png" alt="Overview" width="100%" max-width="800px">
</p>

<!-- Fixed width with fallback -->
<p align="center">
  <picture>
    <source media="(max-width: 600px)" srcset=".github/infographic-mobile.png">
    <img src=".github/readme-infographic.png" alt="Overview" width="800">
  </picture>
</p>
```

## Dark Mode Considerations

GitHub supports dark mode. For infographics:

```markdown
<!-- Different images for light/dark mode -->
<p align="center">
  <picture>
    <source media="(prefers-color-scheme: dark)" srcset=".github/infographic-dark.png">
    <source media="(prefers-color-scheme: light)" srcset=".github/infographic-light.png">
    <img src=".github/readme-infographic.png" alt="Overview" width="800">
  </picture>
</p>
```

## Accessibility

Always include meaningful alt text:

```markdown
<!-- Good -->
<img src="infographic.png" alt="Architecture diagram showing API layer connecting to database and cache services">

<!-- Bad -->
<img src="infographic.png" alt="Diagram">
<img src="infographic.png" alt="infographic.png">
```

## Link Targets

For badges that should open in new tab (external links):

```markdown
<!-- Using HTML for new tab -->
<a href="https://external-link.com" target="_blank">
  <img src="badge-url" alt="Badge">
</a>
```

Note: Standard Markdown links don't support target="_blank", which is generally fine for GitHub READMEs.
